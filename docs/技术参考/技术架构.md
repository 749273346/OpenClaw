# 惠电宝 (Huidianbao) - 技术架构概览

## 1. 系统定位 (System Positioning)
惠电宝是专为铁路电力专业设计的智能安全与作业辅助助手。它基于 OpenClaw 智能体框架，集成了大语言模型（LLM）、检索增强生成（RAG）、多模态识别与企业级即时通讯（IM）接口，旨在解决规程查询难、作业风险高、信息流转慢等痛点。

## 2. 核心架构 (Core Architecture)
系统采用分层微服务架构，确保高内聚、低耦合与可扩展性。

### 2.1 接入层 (Gateway Layer)
- **OpenClaw Gateway**: 处理所有入站与出站流量。
- **IM Connectors**:
  - **WeCom (企业微信)**: 核心交互渠道，支持文本、语音、图片消息。
  - **Feishu (飞书)** / **DingTalk (钉钉)**: 备用及协同办公渠道。
  - **QQ Bot**: 社区与非正式沟通渠道。

### 2.2 调度与编排层 (Orchestration Layer)
- **Agent Runtime**: 基于 OpenClaw 核心，负责会话管理（Session）、上下文维护（Context）与技能调度（Skill Dispatch）。
- **Model Router (模型路由)**:
  - **Fast Track**: 使用 `deepseek-chat` 处理闲聊、简单查询与关键词提取。
  - **Reasoning Track**: 使用 `deepseek-reasoner` 处理复杂规程分析、故障诊断与方案生成。
- **Evolution Loop**: 训练智能体（Trainer Agent）负责自我进化、能力扫描与配置热更新。

### 2.3 能力层 (Skill Layer)
采用插件化架构，所有业务逻辑封装为 Skill：
- **HDB-RAG**: 核心规程检索能力。
  - **Hybrid Search**: 结合 BM25（关键词）与 Embedding（语义）的混合检索。
  - **Zhipu GLM-4V**: 图像识别与多模态分析。
- **System Monitor**: 实时监控 CPU、内存与进程状态。
- **Python/Shell Interpreter**: 安全沙箱内的代码执行环境。

### 2.4 数据与知识层 (Data & Knowledge Layer)
- **Vector Store**: 本地 FAISS/Pickle 索引，存储规程切片的 Embedding 向量。
- **Knowledge Base**: `/root/.openclaw/知识库资料` 下的 Markdown 格式规程文档。
- **Logs & Audit**: 完整的会话日志与进化记录，用于审计与回溯。

## 3. 技术栈 (Tech Stack)
- **Framework**: OpenClaw (Node.js/TypeScript)
- **Language**: Python 3.11 (AI/Data Science), TypeScript (Service/Glue)
- **LLM Provider**: DeepSeek (Reasoning/Chat), Zhipu AI (Embedding/Vision)
- **Storage**: Local FS (RAG Index), JSONL (Sessions)
- **DevOps**: PM2 (Process Management), Git (Version Control)

## 4. 数据流 (Data Flow)
1. **User Query** -> **Gateway** -> **Agent Runtime**
2. **Intent Classification** -> Chat / Work
3. **Work Flow**:
   - -> **Keyword Extraction** & **Embedding Generation**
   - -> **Hybrid Search (BM25 + Vector)** -> **Re-ranking**
   - -> **Context Assembly** -> **LLM Reasoning**
   - -> **Response Formatting (Markdown/Table)**
4. **Response** -> **Gateway** -> **User (IM)**

## 5. 安全与合规 (Security & Compliance)
- **Data Privacy**: 所有敏感数据（API Keys）存储于环境变量或受控配置中。
- **Sandboxing**: 代码执行严格限制在 `/root/.openclaw/workspace` 目录下。
- **Audit**: 关键操作（如文件修改、配置变更）通过 Evolution Log 记录。
